# BibleWorld Benchmark Suite

Run at least 3 of these checks every cycle. Any pattern promoted to build consideration should pass the relevant checks explicitly.

1. Textual Grounding
   Pass if the pattern is anchored in the actual passage and not only in a theme or metaphor.

2. Forced Mapping Rejection
   Pass if the report names at least one candidate mapping that was rejected because the structural match was weak.

3. Big Tech Gap Fit
   Pass if the chosen gap is tied to a named company, product area, or clearly documented developer pain point.

4. Competitor and Novelty Check
   Pass if current tools, repos, papers, or products in the space were checked and the novelty claim was adjusted accordingly.

5. Solo Buildability
   Pass if the cycle explains whether one strong solo builder could ship the proposed tool in 8 weeks and why.
