## NEW BUILD

### BUILD-008: TalentDeploy
**Pattern Source:** Parable of the Talents (Matthew 25:14-30)
**Build Type:** SOFTWARE — Investment Club Performance SaaS
**Problem Solved:** Diaspora investment clubs cannot measure operator performance (IRR), flag idle capital, or make data-driven reallocation decisions
**Who It Serves:** Diaspora investment clubs, angel syndicates, cooperative fund managers
**How It Works:**
- Dashboard tracking IRR per operator/deal (not just absolute profit/loss)
- Deployment rate scoring (% allocated capital actively deployed vs. idle)
- "Buried talent" alerts (capital idle beyond 30/60/90 day thresholds)
- Quarterly Servant classification (Servant 1 = 100%+ return, Servant 2 = positive return, Servant 3 = undeployed/negative)
- Reallocation recommendations based on performance data
- Risk-free rate benchmark (Ghana T-bill rate as floor)
**Claude API Role:** Performance narratives, operator classification, reallocation recommendations, red-flag detection
**Capital Required:** ZERO (laptop + Claude API + Vercel)
**Build Score:** 8.1
**Status:** IN-DESIGN
**Agent Responsible:** Chief Builder + Kingdom Business Director
**Cycle Started:** 003
**Priority:** #3 (after GrantPilot and KnowFirst)
**Synergy:** Integrates with TrustChain (BUILD-005) — TrustChain verifies deals; TalentDeploy tracks performance

### BUILD-005: TrustChain — Design Requirement Added
**From:** PAT-015 science validation (trust decay over chain length)
**Requirement:** Display trust-chain length and decay score. Short chains (2-3 hops) = high trust score. Long chains (5+ hops) = low trust score. Users must be able to filter deals by chain length and minimum trust score.

**From:** Miracle Lab (servant knowledge vs. master ignorance pattern)
**Requirement:** Distinguish between "operator trust" (referrer witnessed the process) and "consumer trust" (referrer only experienced the output). Display trust type alongside trust score.
